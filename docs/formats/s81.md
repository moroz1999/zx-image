# Format: s81 / s80

ZX81 and ZX80 text screens decoded from character token streams.

## Identifiers

| Type key | Plugin class | Font class | Machine |
|----------|-------------|-----------|---------|
| `s81` | `ZxImage\Plugin\S81` | `Zx81FontData` | Sinclair ZX81 |
| `s80` | `ZxImage\Plugin\S80` | `Zx80FontData` | Sinclair ZX80 |

Both extend `Standard`. `S80` extends `S81`, overriding only the font data source.

## Description

These formats render the display memory of ZX81/ZX80 machines where characters are stored as token byte values. Each byte is looked up in the appropriate hardware font and rendered into a 256×192 canvas using the standard ZX color scheme.

## Token Handling

- Tokens 1–63: characters 1–63 in the font table
- Tokens 128–191: same characters as 0–63 (inverse video — mapped by subtracting 64 in the font lookup), rendered identically (no true inversion is performed; the font data stores the inverted form for these ranges in `Zx80FontData`)
- All other tokens: rendered as blank (index 0 = space)

No control codes are processed — every byte is treated as a character token.

## Default Attribute

A fixed attribute `00111000` is applied to all cells:
- Flash = 0, Bright = 0
- Paper = `111` (white)
- Ink = `000` (black)

This matches the ZX81 default white-on-black display (the attribute byte encoding puts paper in bits 5–3 and ink in bits 2–0, so white paper = `111`, black ink = `000`).

## Font Data

### `Zx81FontData`
ZX81 ROM character set. Characters are stored as 8-row × 8-bit binary strings. Includes standard ASCII-like characters plus ZX81-specific block graphics.

### `Zx80FontData`
ZX80 ROM character set. Similar structure to ZX81 but with the older ZX80 character shapes. Additionally stores inverse-video versions of characters (tokens 64–127 map to inverted glyphs stored in indices 0–63 shifted by 64 in the font array).

## Pixel Layout

Same as `Specscii` — characters placed using ZX VRAM's non-linear three-thirds structure.

## Output

Always PNG (`image/png`) — no flash support.
