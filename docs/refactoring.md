Нужно отрефакторить код до современных стандартов стиля и организации архитектуры.

Решаемые проблемы:
1. Код использует многоуровневое наследование с переопределением. Нужно заменить это immutable single purpose сервисами.
2. Код использует array shapes. Все array shapes должны быть заменены на более современные структуры данных, такие как DTO.
3. Код использует стринги для обработки бинарных данных. Не нужно парсить условный char в '00001111', нужно работать с байтами как считаны из оригинала.

Рефакторинг нужно проводить постепенно, выделяя из каждого плагина его функционал в сервисы. В плагинах останется по сути оркестрация, а реализация будет вынесена в сервисы.
Если сервис относится строго к одному плагину, то храни его рядом в том же namespace. Для переиспользуемых сервисов выбирай общий namespace.
После разборки каждого плагина запускай тесты, чтобы убедиться, что все работает как раньше.
Каждый переиспользуемый метод, вынесенный в сервис, помечай как @deprecated. Если легаси-метод больше не используется, удаляй его сразу, но перепроверяй поиском.
